Comandos

¿Recuerdas algunos de los comandos básicos para iniciar, verificar y seguir un repositorio?

Edita esta sección para agregar algunos de los comandos que hemos utilizados en Git y para qué nos ayudan. Puedes agregar más si lo necesitas.


Comandos básicos


git init

Inicializa un repositorio nuevo, puede estar vacio o contener algunos archivos :)

git clone

Te permite clonar un repositorio, es decir, copiar un repositorio ya existente.

git status

Te permite visualizar el status que se encuentra tu repositorio local, si tiene archivos modificados o eliminados

git add

Añade archivos al área de ensayo

git config --list

Permite listar la configuración de git

git log

git-log: muestra los registros de los commit hechos


Comandos para repositorios remotos


git init

Inicializa un repositorio nuevo, puede estar vacio o contener algunos archivos :)

git remote

Lista los repositorios remotos.

git remote add

Con este comando podrás subir tus archivos a la nube.

git push

Te permite subir tus cambios a un repositorio remoto

git pull

Te permite extraer y descargar contenido desde un repositorio remoto y actualizar el repositorio local con el contenido descargado.

git remote show

Te permite ver la información acerca de un remoto en particular.


Comandos para ramas en Git


git push [nombre-remoto] [nombre-rama]

Permite enviar (push) la nueva rama al repositorio remoto.

git checkout + nombre_rama

Con el comando git checkout podrás cambiar de ramas.

git branch -d nombre_rama

Te permite borrar una rama

git merge nombre_rama

Te permite fusionar dos ramas en una sola

git branch nombre_rama

Te permite crear ramas




Recuerda configurar Git por primera vez con los siguientes comandos:

$ git config --global user.name "John Doe"

$ git config --global user.email johndoe@example.com

$ git config --global init.defaultBranch main

Ramas

Trabajar con ramas nos puede ayudar a tener varios ambientes de trabajo, describe por ejemplo que uso le darías a las siguientes ramas.

main/master

La rama master es la rama con la que se comienza en cualquier proyecto, y es la que se utiliza como rama principal donde se encuentra el proyecto en su estado final.

develop/desarrollo

Es la rama donde podemos crear nuevas funcionalidades en el proyecto, siin afectar a la rama productiva

design/diseño

Es la rama donde se pueden editar principalmente las características front-end.

feature/requerimiento

Es una rama separada en su repositorio de Git que se usa para implementar una sola característica en su proyecto.
